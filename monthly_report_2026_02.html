Release Summary

➕ What we added:

  • Advanced Remote Commands (RCZ): Introduced a suite of high-value "On Demand" remote operations, including:

    • Climate Control: Implementation of Remote Climate Requests and Preconditioning Settings.

    • Active Cabin Ventilation (ACV): New on-demand air circulation functionality.

    • Security Features: Added Remote Trunk Lock and Unlock capabilities.

  • Privacy & Data Security: Implemented the Remote Clear Personal Data command, allowing for secure remote wiping of user information from the vehicle.

  • Enhanced Geofencing Precision: Updated the data handling for circular zone triggers. The radius field has been transitioned from an Integer to a Float data type, allowing for much more granular and precise radius definitions in distance-based tracking.

  • VIP Platform Launch: Introduced the Vehicle Driver Interaction Platform (APIM/EAH), including mappers, DAO repositories, and dedicated controller APIs for /changelog and driver interaction persistence.

  • Global & Brand Expansion:

    • F2X South America: Full implementation for FIAT brand activation and regional feature flagging.

    • Legacy Inbound: Specialized handling for FIAT legacy inbound corner cases.

  • Architecture: VDP_STORE_2 (VDS2):

    • Major transition to VDS2, enabling Charging Session storage for all architectures while disabling it in VDS1.

    • Integrated Stolen, Alarm, and Position features into the new storage model.

    • Implemented functional document identifiers and a new TTL (Time-To-Live) strategy defined per FleetID.

  • Smart Data Triggers (PATRIC): Deprecated simple "times" triggers in favor of a sophisticated Schedule-based monitor trigger. Added "times" and "areas" predicates to the monitor event payload for richer context.

  • USB Map Replacement (DOCG): Automated the end-to-end "Real Campaign" flow for USB map updates, including SUPREM prescription fetching, zip link lookup, and target campaign execution.

  • Global Brand Activation (F2X): Completed full implementation for FIAT brand activation in South America, including regional feature flagging and specialized handling for Family Drive Codes.

  • VIP (Vehicle Driver Interaction Platform): Standardized the VIP service layer, including new controller APIs for POST/GET/DELETE operations and a dedicated DAO repository for VIP Change Logs.

  • Advanced Monitoring (PATRIC): * Transitioned from simple "times" triggers to a robust Schedule-based monitoring system.

    • Integrated new triggers for chargePreferenceRate and added "times/areas" predicates to event payloads.

  • J4U / RTBM-V2 Data Expansion: Added raw data conversion and mapping for JDA (RTBM-V2 architectures) to support enhanced vehicle health monitoring.

  • Notification History Evolution: Implemented a high-performance Spring Integration Aggregator (EAH) to handle bulk Notification History upserts via Kafka, reducing database contention.

🔧 What we changed:

  • Communication Integrity: Optimized the feedback loop from DOCG to the Supreme platform during Service Activation to ensure consistent status synchronization.

  • Data Parsing Logic: Optimized the internal parsing engine to ensure all zone trigger circle radii are strictly handled as float types during processing, preventing rounding errors or data loss in geofencing operations.

  • Refined Telemetry Logic: Updated APIM/EAH to prioritize Co (Carbide) data; if unavailable, the system now extracts management details from Cos data.

  • Precise Geofencing: Standardized Zone Trigger circle radius to float across APIM and EAH for improved GPS boundary accuracy.

  • Infrastructure & Security: * Upgraded SDKs to support Java 21 with Mockito java agent configuration.

    • Removed Guava and PowerMock dependencies in favor of plain Mockito to modernize the tech stack.

    • Implemented Error Decoders for Suprem Feign clients to ensure Swagger-compliant error responses.

  • Telemetry Intelligence: Updated processing logic to favor Carbide (Co) source data; the system now includes a new Temperature Module within telemetry responses under the energy category.

  • Geofencing Precision: Zone Trigger radii have been updated to float across APIM/EAH/PATRIC to ensure GPS accuracy for circular boundaries.

  • Infrastructure Modernization:

    • Removed Guava and PowerMock in favor of native Mockito and Java 21 standards.

    • Implemented Error Decoders for the Suprem Feign client to ensure all error responses align with Swagger specifications.

  • Maintenance Tracking: Refactored maintenance data (days/mileage before maintenance) to be handled as timestamped telemetry entries.

🔨 What we fixed:

  • Message Consumption: Resolved an issue in the Pre-Production environment where Lifecycle messages were not being properly consumed by the DOCG gateway.

  • Inventory Accuracy: Patched a bug in Post-Operational Inventory management that caused redundant or multiple inventories to be processed/sent for a single operation.

  • Supreme Integration: Fixed the Service Activation flow to ensure feedback is correctly delivered to Supreme, preventing "stuck" activation statuses.

  • System Stability: Resolved a critical NullPointerException (NPE) within the ExtensionEventMapper. This fix ensures that incoming extension events are mapped correctly without causing service interruptions, especially when encountering unexpected or null data fields.

  • Data Mapping & Visibility:

    • Fixed missing Driving Mode stats and battery temperature blocks for Carbide architectures.

    • Corrected the "lifetime mileage" mapping discrepancies between Carbide and VDP sampling.

    • Restored Stolen Information in the telemetries/latest endpoint.

  • DOCG Campaign Stability:

    • Resolved the "Submit" button failure for specific ScomoID natures.

    • Patched the ESA key issue causing VSM campaign installation failures.

    • Fixed the "multiple inventories" bug where redundant Post-OP inventory messages were sent to CMM.

  • API Reliability: Patched the random 404/500 errors affecting B2B/B2C APIs and corrected the timestamp query parameter logic for the VIP platform.

  • Data Mapping & Integrity: * Fixed the "missing drivingMode" bug in Trip statistics.

    • Resolved critical mapping issues where fuelLevel and battery.health (capacity/resistance) were failing to populate from Carbide sources.

    • Corrected the tyreState enum and the vehicle.engines.type naming convention.

  • Operational Stability:

    • Resolved the "multiple inventories" bug in DOCG Post-Op flows.

    • Fixed random 404/500 errors across B2B/B2C APIs caused by incorrect container orchestration.

    • Patched an issue where "Previous Trips" were not being closed correctly in VDP_PROC.
